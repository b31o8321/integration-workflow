# Zenoti 接入可行性评估

> 分析日期: 2026-04-09
> 分析人: Claude (intelli:analyze)
> 数据来源: https://docs.zenoti.com/docs/overview + https://developer.zenoti.com/reference
> 业务场景: 美业预约系统，AI 语音对话中查询/创建/修改/取消预约

---

## 一、结论（PM / 交付用）

| 功能 | 结论 | 说明 |
|------|------|------|
| 工单AI回复 | ❌ 不可行 | 预约管理平台，无工单系统，不适用 |
| Livechat对接 | ❌ 不可行 | 无实时消息通道，不适用 |
| 数据同步（预约/客户读取） | ⚠️ 部分可行（需适配） | 预约和客户数据可同步；无 `updated_after`，需 Webhook + 滚动窗口双轨；速率上限未公开 |
| **AI 预约写入（新）** | ✅ 可行 | 创建/改期/取消/查询可用时段四项 API 完整，是该平台最高价值能力 |

### 建议

Zenoti 是整个美业 AI 链路的**预约落单层**，建议作为必接平台。优先实现 AI 预约写入 tool（创建/查询时段/取消/改期），这是 AI 语音引导预约的终态操作；数据同步层采用 Webhook 为主、轮询兜底的双轨策略，接入成本可控。工单和 Livechat 维度不适用，无需投入。

---

## 二、技术差距分析（研发用）

### 工单AI回复

**可行性：** ❌ 不可行

Zenoti 无工单/客服系统，不建议实现。

---

### Livechat对接

**可行性：** ❌ 不可行

Zenoti 无实时消息通道，不建议实现。

---

### 数据同步（预约/客户读取）

**可行性：** ⚠️ 部分可行

**差距列表：**

- [Minor] **无 `updated_after` 参数** — 预约和客户 API 仅支持日期范围过滤（`start_date`/`end_date`），无法精准拉取变更记录。需采用滚动时间窗口（如过去 24 小时）轮询，并以 Webhook 为主要增量通道。增加 API 调用量，但不阻断功能。
- [Minor] **速率上限未公开** — 限速数字未在文档中明确，需生产前压测确认容量上限。以 HTTP 429 + `Retry-After` 头为退避信号，加指数退避策略。
- [Minor] **客户无增量参数** — `GET /v1/guests` 无变更时间过滤；建议以每日全量同步为主（客户记录变更频率低），Webhook 中的 guest_id 关联变更为辅助补充。

**预估工作量：** 小（2–3 天，含 ISyncService 实现 + Webhook 注册 + 退避策略）

---

### AI 预约写入（新维度）

**可行性：** ✅ 可行

**差距列表：**

- [Minor] **创建前必须查询可用时段** — `POST /appointments` 需提供具体的 therapist_id 和 slot；必须先调用 `GET /centers/{id}/slots` 确认可用性，不可跳过，否则有双重预订风险。已在 AI 对话流程中规划为必要步骤。
- [Minor] **速率上限同上** — 高频预约创建场景（如促销期）需压测确认。

**预估工作量：** 小（3–5 天，含 4 个 tool 实现 + 对话状态集成 + 防误操作确认机制）

## 三、接入 Checklist（研发用）

### 数据同步接入 Checklist（预约/客户读取）

- [ ] 注册 Zenoti Webhook（`appointment.created`、`appointment.updated`、`appointment.cancelled`）
  - [ ] 配置 callback endpoint
  - [ ] 实现 `X-Zenoti-Signature` HMAC 签名验证
- [ ] 实现 `ISyncService` 预约初始全量同步
  - [ ] `GET /v1/centers/{id}/appointments`（按中心 + 日期分片）
  - [ ] 分页：`page` + `size=100`
  - [ ] 写入 Intelli 预约缓存（appointment store）
- [ ] 实现增量维护（双轨）
  - [ ] 主轨：Webhook 实时更新缓存（appointment.created/updated/cancelled）
  - [ ] 兜底：每 15 分钟滚动扫描过去 24 小时（捕获 Webhook 漏事件）
- [ ] 实现 `ISyncService` 客户数据同步
  - [ ] `GET /v1/guests`（每日全量同步）
  - [ ] Webhook 中 guest_id 变更触发单条更新
- [ ] Rate limit 处理：指数退避 + `Retry-After` 头解析

### AI 预约写入 Checklist（新 Action Tool 层）

- [ ] 实现 `checkAvailability` tool
  - [ ] 调用：`GET /v1/centers/{id}/slots`
  - [ ] 参数：center_id、service_id、date
  - [ ] 返回：可用时段列表（供 AI 向客户展示选项）
- [ ] 实现 `createAppointment` tool
  - [ ] 调用：`POST /v1/appointments`
  - [ ] 必填：center_id、service_id、guest_id、therapist_id、start_time
  - [ ] 前置：必须先调用 `checkAvailability` 确认时段
  - [ ] 前置：必须获得客户明确确认（防误操作门控）
  - [ ] 成功后：立即更新 Intelli 本地缓存
- [ ] 实现 `rescheduleAppointment` tool
  - [ ] 调用：`PUT /v1/appointments/{id}`
  - [ ] 前置：同 createAppointment（查时段 + 客户确认）
- [ ] 实现 `cancelAppointment` tool
  - [ ] 调用：`POST /v1/appointments/{id}/cancel`
  - [ ] 前置：客户明确确认取消意图
- [ ] 将四个 tool 注册到 AI Voice Engine / 对话管理器
- [ ] 实现对话中的预约信息收集状态机
  - [ ] 收集：服务类型 → 门店 → 日期时间偏好
  - [ ] 调用 checkAvailability → 向客户展示可用时段
  - [ ] 客户选择 + 确认 → 调用 createAppointment
